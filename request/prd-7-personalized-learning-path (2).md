## 1. 文档概述

### 1.1 模块定位
本模块是“AI驱动的多智能体学习系统”的大脑中枢，承担**自适应学习路径规划**与**多模态资源主动推荐**两大职责。它不依赖学生主动搜索，而是由系统根据动态学习画像、知识掌握状态和科目目标，**自动预测并推送**学生当下最需要的学习内容。

### 1.2 核心价值
- **主动供给**：系统无缝嵌入学习流，在恰当的时机自动呈现资源，实现“零搜索学习”。
- **一人一路径**：每个学生的路径都基于其专属6维画像和目标实时动态生成。
- **本地轻量运行**：所有核心算法可在普通笔记本上实时计算，无需依赖GPU或云端大模型。
- **可视化可控**：学生可随时查看、调整学习计划，但系统默认自动优化。

### 1.3 术语定义
- **KG（知识图谱）**：学科知识点及其前驱后继关系的有向图。
- **掌握度**：学生对某知识点的掌握概率，取值0~1。
- **学习画像**：包含知识基础、认知风格、易错点偏好等不少于6维的动态学生特征向量。
- **学习路径**：按知识图谱拓扑及掌握度规划的一系列学习节点与推荐动作序列。
- **自适应引擎**：结合轻量知识追踪、目标差距分析和多Agent协同决策，实现路径实时调整的核心算法集。

## 2. 用户场景与核心需求

### 2.1 典型场景
- **初次学习**：学生添加“离散数学”，设定目标“期末90分”。系统通过对话诊断定位薄弱点，自动生成学习路径，并立即推送第一章的讲解视频和练习。
- **练习纠偏**：学生完成一组命题逻辑练习题后，系统根据错题自动从底部滑出针对性强化面板，推荐变式题和错因图解。
- **路径动态更新**：学习过程中，学生觉得某章已掌握，手动标记“跳过”，路径立刻重排，后续节点和资源同步刷新。
- **目标预警**：阶段评估发现当前进度仅能达78分，系统主动弹出调整建议，插入补救节点，学生可一键接受。

### 2.2 核心需求
- 我想设定一个具体的学习目标，剩下的由系统自动规划。
- 我需要的内容应该在需要的时候**自动出现**，而不是我去找。
- 我需要一个能看见、能理解、能调整的学习路径图。
- 系统推荐资源时必须告诉我**为什么推荐**，让我信任它。
- 全部功能要在我的笔记本电脑上流畅运行。

## 3. 系统架构与多智能体设计

### 3.1 整体架构
```text
┌─────────────┐  ┌──────────────┐  ┌───────────────┐
│  学习画像    │  │  知识图谱    │  │  资源池       │
│  (6维动态)   │  │  (KG)       │  │  (多模态)     │
└──────┬──────┘  └──────┬───────┘  └───────┬───────┘
       │                │                  │
       └────────┬───────┴──────────────────┘
                │
     ┌──────────▼──────────────────────────┐
     │         自适应学习路径引擎           │
     │  ┌──────────────────────────────┐   │
     │  │       多智能体协同层          │   │
     │  │  · 诊断Agent                 │   │
     │  │  · 路径规划Agent              │   │
     │  │  · 资源匹配Agent              │   │
     │  │  · 推送决策Agent              │   │
     │  │  · 评估调控Agent              │   │
     │  └──────────────────────────────┘   │
     │  算法核心：BKT + 贪心规则 + 混合推荐  │
     └──────────┬──────────────────────────┘
                │
     ┌──────────▼──────────────────────────┐
     │         学习交互前端                │
     │  (路径看板、资源卡片流、强化面板)     │
     └─────────────────────────────────────┘
```

### 3.2 多智能体协同机制
- **画像分析Agent**：读取数据库中的6维动态画像，输出特征向量，供其他Agent调用。
- **诊断Agent**：基于答题数据和贝叶斯知识追踪（BKT）实时更新各知识点掌握概率。
- **路径规划Agent**：结合目标、知识图谱和当前掌握度，采用贪心+规则算法输出下一步最佳（节点，动作类型）。
- **资源匹配Agent**：接收（知识点，动作类型，画像偏好），执行内容+协同混合推荐，返回排序后资源包。
- **推送决策Agent**：判断推荐时机、展示形式和打断策略，将资源封装为前端卡片。
- **评估调控Agent**：周期性预测期末得分，若偏离目标则插入补救节点并通知路径重规划。

**协同流程示例**：
```
学生完成答题 → 诊断Agent更新掌握度 →
路径规划Agent检测到节点A已达标，锁定下一节点B(动作:视频讲解) →
资源匹配Agent结合画像(视觉型+易混淆)检索“对比动画视频” →
推送决策Agent判断当前无深度沉浸任务，在主界面右侧推入新学习卡片。
```

## 4. 关键设计：自适应引擎与主动推荐

### 4.1 自适应引擎如何工作
自适应 = **实时诊断 + 动态规划 + 偏好匹配 + 目标纠偏** 的闭环，具体分为四层：

1. **状态感知层**：采用轻量**贝叶斯知识追踪（BKT）**，每个知识点仅存储4个参数（初始掌握、猜测、滑落、学习率），答题后即时更新掌握度，CPU占用可忽略。
2. **规划决策层**：基于知识图谱的贪心算法。候选集为所有前驱已完成的未完成节点，计算节点得分 `Score = w1*(1-掌握度) + w2*重要度 + w3*考察频率 - w4*认知负荷惩罚`，选出最高分节点。动作决策依据：掌握度<0.3推“概念讲解”，0.3~0.7推“题库练习”，>0.7推“实操案例”。
3. **资源匹配层**：离线将全部资源用轻量嵌入模型（MiniLM-v2，100MB）转为向量；在线用画像偏好向量与资源向量做点积相似度计算，结合协同过滤（预计算相似学生群组）排序，筛选Top3。
4. **动态调整层**：微循环（单次学习后）立即更新掌握度并解锁后续节点；宏循环（章节完成或定时）由评估Agent预测期末得分，若低于目标，自动插入补救节点并调整难度曲线。

### 4.2 系统在何时主动推荐
所有推荐均由事件驱动，**不依赖学生搜索**，具体触发时机与交互方式如下：

| 触发时机 | 触发条件 | 推荐内容 | 前端展示方式 |
|----------|----------|----------|--------------|
| **1. 路径节点激活** | 进入科目、上一节点完成、接受评估调整 | 新节点的讲解视频/文档/预习题等资源包 | 右侧卡片流自动推入，对应节点高亮 |
| **2. 练习/测试后** | 学生提交一组练习题 | 基于错题知识点的纠错视频、变式题、图解 | 底部滑出强化面板，可下拉收起 |
| **3. 掌握度突破阈值** | 某知识点掌握度从<0.8变为≥0.8 | 解锁后继节点资源，拓展阅读 | 路径看板节点解锁动画 + 轻提示，不打断主任务 |
| **4. 学习画像更新** | 系统检测到认知风格、易错偏好变化 | 调整资源类型占比（如文档→视频） | 无感切换，下次节点资源自动适配 |
| **5. 阶段评估后** | 完成章节或每周定时评估，预测得分偏离目标 | 调整后的学习计划，补救资源包 | 非阻断模态弹窗，展示差距并提供“接受/保持原计划”选项 |
| **6. 学生主动干预** | 拖拽路径、输入意图（如“我想多练证明题”） | 重规划后的新路径及配套资源 | 路径刷新动画，新节点列表即时呈现 |

**打断原则**：系统会检测学生是否处于深度专注状态（如视频全屏、做题中），若是，则高优先级推送（如阶段评估）改为通知栏提示，低优先级推送延迟至空闲时展现。

## 5. 前端功能设计（React + TypeScript）

### 5.1 技术选型与本地性能
- **框架**：React 18 + TypeScript + Vite
- **状态管理**：Zustand（轻量）
- **动画**：framer-motion（硬件加速，体积小）
- **可视化**：@antv/g6（知识图谱按需加载）
- **网络请求**：axios + @tanstack/react-query（缓存与自动重取）
- **打包后大小**：gzip <200KB，内存占用典型<100MB

### 5.2 主要界面组件

#### 5.2.1 学习主页面 `<SubjectLearningPage>`
- **顶部 `<GoalProgressHeader>`**：显示科目目标、当前进度条与预估分数。
- **左侧 `<PathBoard>`**：垂直时间轴路径看板。
  - 每个节点为卡片，包含名称、动作图标、掌握度圆环、状态标签。
  - 激活节点有呼吸光环动画，锁定节点显示锁图标。
  - 支持拖拽排序（@dnd-kit）、右键菜单（重学/跳过/标记已会）。
  - 动态更新：节点插入/删除/解锁均有layout动画。
- **右侧 `<ContentStream>`**：主动推送资源卡片流。
  - 监听当前激活节点的资源包，变化时旧卡片向左退出，新卡片从右滑入。
  - 卡片内含Tabs切换：视频、文档、练习、拓展阅读。
  - 每个资源项展示缩略图、类型、时长，可点击开始学习。
  - 底部“为什么推荐？”按钮展开画像理由。
- **底部 `<IntentInput>`**：自然语言意图输入框及快捷标签。
- **浮层 `<DiagnosticPanel>`**：练习后从底部滑出，高度40%，展示错题强化资源，可下滑关闭。
- **弹窗 `<StageEvaluationModal>`**：阶段评估结果，非阻断式，可稍后处理。

#### 5.2.2 路径全屏视图 `<PathFullScreen>`
- 使用G6渲染知识图谱，节点大小表示重要度，颜色表示掌握度，连线表示前驱关系。
- 悬停节点显示“建议从这里继续”气泡，薄弱章节节点闪烁橙光。

### 5.3 主动推荐的前端实现流
- **无刷新推送**：使用 `react-query` 的 `useQuery` 定期拉取激活节点和资源包（可配合窗口聚焦事件），或用轮询检测状态变化。
- **事件响应**：
  - 练习提交 → 调用 `submitAnswer` 接口，返回错题分析后设置 `showDiagnosticPanel=true`。
  - 意图输入/拖拽 → 调用 `applyIntent`，返回新路径和资源包，直接更新状态。
  - 阶段评估 → `useQuery` 设置10分钟轮询，检测到未读评估则显示模态框。
- **沉浸模式检测**：通过监控视频全屏、表单焦点等状态，决定推送是直接展示还是收纳至通知中心。

## 6. 后端算法与轻量实现方案

### 6.1 贝叶斯知识追踪 (BKT)
- 每个知识点维护四个参数：`P(L0)` 初始掌握概率，`P(G)` 猜测概率，`P(S)` 滑落概率，`P(T)` 学习率。
- 答对/答错后使用贝叶斯公式更新后验掌握概率。运算仅涉及乘法和加法，单次更新<1ms。

### 6.2 路径规划贪心算法
- 输入：KG、各节点掌握度、目标权重。
- 构建候选集：所有前驱节点已完成的未完成节点。
- 计算得分：`Score = a*(1-mastery) + b*importance + c*exam_freq - d*cognitive_load`。
- 选取最高分节点，动作类型由掌握度规则确定。

### 6.3 混合推荐引擎
- **离线部分**：将所有资源（文本摘要）用MiniLM-v2模型转为512维向量，存为numpy数组；构建学生协同相似索引。
- **在线部分**：
  1. 召回：知识节点精确召回 + 画像兴趣召回 + 协同召回 + 错题变式召回。
  2. 排序：加权公式（难度匹配度、风格契合度、质量评分、薄弱点覆盖度）筛选Top3。
  3. 多样性控制：至少包含讲解和练习两类资源，5%探索性资源。
- **性能**：5000条资源内，单次检索<50ms，内存占用<50MB。

### 6.4 数据库与存储
- **SQLite**存储：学习画像、知识图谱、掌握度状态、路径节点、资源元数据。
- **内存缓存**：当前活跃学生的画像向量、资源向量，加速推荐计算。

## 7. 数据流与API接口

### 7.1 核心数据流
```
行为日志 → 画像Agent(更新画像) + 诊断Agent(更新掌握度)
→ 路径规划Agent(读取KG、画像、掌握度) → 输出激活节点
→ 资源匹配Agent(结合画像偏好) → 返回资源包
→ 推送决策Agent(时机检测) → 前端展示。
```

### 7.2 关键API
| 接口 | 方法 | 说明 | 请求/响应核心字段 |
|------|------|------|------------------|
| `/api/path/active-node` | GET | 获取当前激活节点及资源包 | `subjectId` → `PathNode`, `ResourceBundle` |
| `/api/practice/submit` | POST | 提交答案，返回掌握度更新和错题强化资源 | `answers[]` → `masteryUpdates`, `diagnosticResources` |
| `/api/path/intent` | POST | 处理用户意图（拖拽/文本） | `subjectId, intent` → `PathNode[]`, `ResourceBundle` |
| `/api/eval/latest` | GET | 获取最新阶段评估结果 | `subjectId` → `StageEval` |
| `/api/path/adjust` | POST | 接受评估调整建议 | `evalId, action` → 新路径 |

## 8. 非功能性需求

- **性能**：答题后掌握度更新延迟<1s，路径重规划<3s，资源检索<200ms。
- **资源占用**：后端服务内存<200MB，前端页面内存<100MB，磁盘占用含模型<1GB。
- **可解释性**：所有推荐资源均可展示推荐理由（知识薄弱点、画像偏好、协同依据）。
- **可扩展性**：新增科目只需构建知识图谱并导入资源池，算法层无需改动。
- **安全性**：本地运行数据不外传（除可选的远程LLM调用）。

## 9. 附录

### 9.1 开发任务建议拆分
- **后端Agent组**：实现BKT、贪心路径规划、混合推荐、评估调控。
- **算法组**：资源向量化与离线索引构建、协同相似矩阵预计算。

### 9.2 比赛展示重点
- 演示从设定目标到自动推送资源的完整闭环，强调无搜索操作。
- 重点展现路径看板的实时更新动画与资源主动推入效果。
- 展示一次错题后的强化面板滑出，以及阶段评估后的计划调整弹窗。
